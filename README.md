This repository represents a collaborative endeavor aimed at upholding the Open Source Definition (OSD), a set of criteria that software must meet to qualify as open source. In the past, the Open Source Initiative (OSI) corporation was entrusted with the responsibility of maintaining the OSD, a task it [struggled to fulfill](https://collectivesource.org/the-open-source-initiative). After about two decades of stagnation, the removal of vendor lock-in from the Open Source Definition marks a milestone, empowering more developers to embrace modern open source licenses like SSPL for the benefit of our community. If you have suggestions or comments to contribute, please share them by submitting an issue. The extent and significance of your involvement determine the responsibilities you will be entrusted with by the vote of the community.

Although the OSI did not maintain the history log of the OSD releases on their website, we were able to retrieve a copy of some of the previous versions and their change logs from third-party public archives.

The following is a **draft** of the version 1.11 (unofficially known as "the release of 2024"), representing the most significant improvement to the OSD since 1997. Notably, clauses #5 and #6 are now properly phrased to convey their intended messages and to guarantee the freedom to protect open source software in a modern market environment.

This version is expected to be published to [CollectiveSource.org](https://collectivesource.org) soon. If you would like to discuss any changes to the draft or have suggestions or comments, please submit an issue.

### Introduction

Open source refers to a category of software licenses that grant individuals and organizations the freedom to use, modify, distribute, and share software. The software’s source code is made available to the public, enabling transparency, collaboration, and community involvement in its development.

To be considered open source, a software license must adhere to the following principles:

### 1. Unrestricted Distribution
The software license must allow the unrestricted free or paid distribution of the software, its source code, and any derivative works under the original terms and without imposing any licensing fees or other charges.

### 2. Source Code
The software’s source code must be unconditionally accessible to the public and should not be made obfuscated or otherwise difficult to study.

### 3. Derivative Works
The license should permit modifications of the licensed material and the creation of derivative works.

### 4. Integrity of The Author’s Source Code
The license may restrict the source code from being distributed in a modified form only if the license allows the distribution of patch files alongside the source code for the purpose of modifying the program at build time. The license must explicitly permit the distribution of software built from modified source code. The license may require derived works to carry a different name or version number from the original software.

### 5. No Prejudice Against Persons or Groups
The license is not allowed to treat any licensee differently in any way other than by making use of tiered pricing based on the entity’s type (individual or corporation), scale (e.g. number of employees), and/or tax classification (e.g. non-profit).

### 6. No Prejudice Against Fields of Endeavor
The license may prohibit open-source hijacking efforts, where an entity diverts revenue from the project by utilizing the software in a competing closed-source commercial product or service. All other fields of endeavor must be treated equally.

### 7. Distribution of License
The rights associated with the program should extend to all individuals who receive the program, without requiring them to execute an additional license.

### 8. License Must Not Be Specific to a Product
The program’s rights should not depend on its inclusion in a specific software distribution. If the program is extracted from that distribution and used or distributed according to the program’s license terms, all recipients of the program should possess the same rights as those granted in connection with the original software distribution.

### 9. License Must Not Restrict Other Software
The license must not place restrictions on other software that is distributed alongside the licensed software. For example, the license must not insist that all other programs distributed on the same medium must be open-source software.

### 10. License Must Be Technology-Neutral
The license must not contain any provisions that are based on specific technologies or styles of interfaces.
